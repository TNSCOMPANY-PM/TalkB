## 🎯 작업 배경 (회의 결정사항)

### 핵심 진단
- 메인 페이지에 텍스트가 너무 많고 부연 설명·중복 섹션이 무거움
- 비용 관련 섹션이 3개로 분산되어 있어 통합 필요
- 작동 원리·점유율 통계는 메인에 있을 필요 없음 → 서브로 이동
- 같은 내용을 단어만 바꿔 반복하는 섹션 多 → 정리하면 분량 절반 이상 감소 가능

### 타겟 우선순위 (메인 메시지 기준)
- **1순위**: 자영업자 (요식업 사장님)
- **2순위**: 본사 (가맹점주용 → 예비 창업자용 순)
- 예비 창업자 관련 콘텐츠는 메시지 흐림 방지를 위해 **별도 서브 탭**으로 분리

## 🏗️ 메인 페이지 신규 8단 구조 (확정)

```
1. Hero          — 강한 한 줄 카피 (불안감 조성)
2. 왜 토크비인가
3. 우리가 해줄 수 있는 것
4. 왜 지금 해야 하는가  (네이버 / SNS / AI 마케팅 비교 + "저점" 메시지)
5. 가격 플랜      ← 기존 3개 비용 섹션 → 1~2개로 통합
6. FAQ
7. CTA           — 무료 진단
8. 콘텐츠         — 썸네일 + 가로 스크롤
```

### 메인에서 빠져야 할 항목 (서브로 이동)
- AI 점유율 통계
- 작동 원리 (How it works)
- 최적화 방식
- "고객 확보 방식에 따라 결과가 다릅니다" 같은 원리 설명 섹션
- 인사이트/지표 상세 자료

→ **별도 "자료·인사이트" 페이지** (`/resources.html` 또는 기존 insights/ 활용) 로 모음

### 메인에 유지하되 보완할 것
- 전문 용어는 메인에 남기되 **툴팁으로 보완** (현재 툴팁 정상 작동 중 — 그대로 유지)
- 신뢰 배지 4종 (9년차 / MAIN-BIZ / ISO / 특허) — Hero 직후 위치 유지

## 📝 카피 결정사항

### Hero 카피 3안 (사용자 선정 필요 — Phase 5에서 처리)
1. "AI는 당신 옆집을 추천하고 있습니다."
2. "네이버에 노출돼 있어도 AI 답변에 우리 매장이 없으면 손님은 옆집으로 갑니다."
3. "GPT에 검색하면 당신 옆집만 나옵니다."

→ **Phase 5 진입 전 사용자(김태정 대표)가 1안 선택**. 미선정 상태면 임시로 1안 사용 후 추후 교체.

### 보조 카피 (확정)
- **"AI 마케팅, 지금이 저점입니다."** → 메인의 "왜 지금 해야 하는가" 섹션 핵심 카피로 활용

### 타이틀 (확정)
- "넘버원 요식업 전문 AI 노출 솔루션" → **유지**
- "1세대" 표현은 부정 뉘앙스라 **제외** (이미 빠져 있다면 그대로 두기)

## 🛠️ 작업 Phase (순서 엄수)

### Phase 0 · 백업 및 현황 분석 [선행 필수]
1. 현재 작업 브랜치 확인: `git status` 깨끗한지 확인
2. **백업 브랜치 생성**: `git checkout -b backup/pre-revamp-2026-04-28` 후 다시 main으로 복귀
   → 만약 잘못되면 이 브랜치로 복원 가능
3. `index.html` 현재 섹션 구조를 분석해서 **다음 항목을 표로 정리해 사용자에게 보고**:
   - 현재 섹션 목록 (위에서 아래 순서대로)
   - 각 섹션별 라인 수 추정
   - 8단 신규 구조에 어떤 섹션을 매핑할지
   - 서브로 이동할 섹션 목록
   - 삭제·통합할 비용 관련 섹션 3개의 위치
4. **여기서 멈추고 사용자 확인 받기**. 매핑 표가 OK 떨어지면 Phase 1 진행.

---

### Phase 1 · 메인 페이지 8단 골격 재배치
1. `index.html`의 섹션 순서를 위 8단 구조에 맞게 재정렬
2. 이때 **각 섹션 내용은 최대한 보존**, 순서만 변경 + 명백한 중복만 제거
3. 큰 변경 사항은 주석으로 표시: `<!-- REVAMP 2026-04-28: 섹션 재배치 -->`
4. 네비게이션 메뉴도 새 구조에 맞게 업데이트
   (현재: AI 검색 · 왜 지금? · 작동 방식 · 가격 · 시작 방법 · 자료 · 무료 진단)
   → 신규 후보: **왜 토크비 · 서비스 · 왜 지금 · 가격 · FAQ · 자료 · 무료 진단**
5. **여기서 멈추고 결과 보여주기**. 사용자 OK 후 Phase 2 진행.

---

### Phase 2 · 부연 설명·원리 섹션 서브 페이지 분리
1. 분리 대상 섹션을 확정해서 사용자에게 한 번 더 확인 (Phase 0에서 정리한 목록)
2. 새 페이지 생성: **`/resources.html`** (또는 사용자 결정 후 다른 이름)
   - 같은 디자인 시스템·폰트·신뢰 배지 사용
   - 섹션 구성: AI 점유율 통계 / 작동 원리 / 최적화 방식 / "고객 확보 방식..." 원리 섹션 등
3. 기존 메인의 해당 섹션 **삭제** + 메인 적절한 곳에 **"자세히 보기 →"** 링크로 `/resources.html` 안내
4. 네비게이션 "자료" 메뉴를 `/resources.html`에 연결
5. `sitemap.xml`에 새 URL 추가
6. `llms.txt`에 새 페이지 항목 추가
7. **여기서 멈추고 결과 보여주기**.

---

### Phase 3 · 비용 섹션 3개 → 1~2개로 통합
1. 현재 비용 관련 섹션 3개의 위치·내용을 표로 정리
2. 통합 방식 제안 (예: "가격 카드 4단(Free/Insights/Solution/Enterprise) + FAQ 안 가격 관련 Q&A 1개"의 2개 구조)
3. 사용자 OK 후 통합 실행
4. 가격 정보 4곳 동기화 검증 (JSON-LD Service / FAQ / 가격 카드 / sitemap)
5. **여기서 멈추고 결과 보여주기**.

---

### Phase 4 · 콘텐츠 영역 가로 스크롤 썸네일로 변경
1. 현재 콘텐츠 영역(인사이트 11편 노출 부분) 위치 확인
2. 가로 스크롤 카드 컴포넌트로 변경:
   - 썸네일(OG 이미지 활용 가능: `og-insights-*.png`)
   - 카테고리 태그
   - 제목 (1~2줄)
   - 좌우 화살표 또는 스와이프
3. 기존 OG 이미지 12개 재활용해 별도 썸네일 제작 부담 제로
4. 모바일 터치 스와이프 지원 (44px 터치 타겟 유지)
5. **여기서 멈추고 결과 보여주기**.

---

### Phase 5 · Hero 카피 적용 [사용자 결정 대기]
1. 사용자(김태정 대표)에게 Hero 카피 3안 중 선택 받기
   - 미선정 시: **1안("AI는 당신 옆집을 추천하고 있습니다.")으로 임시 적용** 후 추후 교체 안내
2. 선정 카피로 Hero 섹션 교체
3. 보조 카피 **"AI 마케팅, 지금이 저점입니다."** 를 "왜 지금 해야 하는가" 섹션 헤드라인으로 적용
4. **여기서 멈추고 결과 보여주기**.

---

### Phase 6 · 종합 검증 및 배포
검증 항목 (Step 8 검증과 동일 기준):
- [ ] HTML 태그 무결성 (모든 파일)
- [ ] JSON-LD 유효성 (특히 가격 정보)
- [ ] JavaScript syntax
- [ ] 구 가격(20/35/60만원) 잔존 0건 — 신규로 들어간 부분에서도 확인
- [ ] 구 브랜드(Apixo) 잔존 0건
- [ ] 사업자 정보 일관성
- [ ] 가격 정보 4곳 동기화
- [ ] sitemap.xml URL (메인 + insights 11편 + resources 1개 + 기타 = 13개 이상 예상)
- [ ] llms.txt 인사이트 11편 + resources 페이지 반영
- [ ] 작성자 표기 통일 (TalkB · 김태정)
- [ ] 모바일 터치 타겟 44px
- [ ] 네비게이션 → 새 섹션 스크롤 정상
- [ ] 가로 스크롤 썸네일 정상 작동 (PC + 모바일)
- [ ] /resources.html 정상 노출
- [ ] 구 URL → 신규 URL 301 리다이렉트 (필요 시 vercel.json 추가)

검증 통과 후:
1. `git add .`
2. `git commit -m "Site Revamp: 메인 8단 구조 재배치 + 서브 분리 + 비용 통합 + 가로 스크롤"`
3. `git push`
4. Vercel 자동 배포 (1~2분) 후 `https://talkb.vercel.app/` 확인

### 무료 진단 라이브 출력 구조 [연구 단계]

- 구현 방식 연구가 먼저 필요하므로 이번 Phase에는 포함하지 않음
- Claude Code 작업 시 **이 부분은 건드리지 말 것** (현재 mailto 구조 유지)

### 본사용 플랜 분리 (예비 창업자 / 가맹점주)

- 예비 창업자용 / 브랜딩 서비스: 신뢰 쌓인 후
- 예비 창업자 콘텐츠 → 별도 서브 탭 (메인 메시지 흐림 방지)
